# Skills Catalog

One row per skill. Scan the **Summary** column for the `Use when …`
clause matching the user's request, then open the linked `SKILL.md`.
The **Trigger keywords** column comes straight from each skill's
`description` frontmatter so a routing agent can keyword-match
synonyms (CLI names, container tags, class names, repo URL fragments).

The **Version** column carries the skill's frontmatter `version:`
SemVer string. The **Source** column shows the upstream
`<branch>@<short-sha>` the canonical `SKILL.md` symlink resolves to
(blank for orphan decorations authored by hand). Every slug-level
`<slug>/SKILL.md` is a stable, agentskills.io-compliant entry point;
for tools it is a relative symlink into the matching
`<slug>/_versions/<branch>/<sha>/SKILL.md` versioned directory
underneath, and sibling `references/`, `scripts/`, and `assets/`
symlinks resolve to the same versioned snapshot.

The skills directory is the canonical location per NV "Skills as
Docs" Standard §2. On checkouts that have completed the §2 migration
the canonical path is `<repo>/skills/`, and `.agents/skills/` plus
`.claude/skills/` are discovery symlinks pointing here. On
un-migrated checkouts (no top-level `skills/`), `.agents/skills/` IS
the canonical source, and `.claude/skills/` is a relative symlink
pointing at it (`.claude/skills -> ../.agents/skills`); the
orchestrator creates that symlink on every run so Cursor / NemoClaw /
Claude Code / Codex resolve the same content without duplication.

Hand-authored routing / index skills that live at the canonical
skills directory root (e.g. `<skills_dir>/SKILL.md` for `nurec-index`)
are included in this catalog alongside the per-slug LLM-generated
entries. They appear with a blank **Source** column because they
have no upstream branch / commit to track.
